# protoc-gen-go-sr

A Protocol Buffers compiler plugin that generates self-contained Schema Registry schemas as Go constants.

For each message annotated with `(redpanda.api.common.v1.schema_registry)`, the plugin generates:

- A **self-contained `.proto` string** with the target message at index 0, followed by its transitive dependencies. Services, validation annotations, and HTTP annotations are stripped. Well-known types (`google/protobuf/*`) are kept as imports.
- A **subject name** for Schema Registry registration.

Both are emitted as Go string constants in a `_sr.go` file alongside the standard generated code.

## Why

Storing protobuf in Kafka with Schema Registry requires registering the `.proto` schema and encoding payloads with the [Confluent wire format](https://docs.confluent.io/platform/current/schema-registry/fundamentals/serdes-develop/index.html#wire-format): `[0x00][schema_id][message_index][protobuf_bytes]`.

The message index is the position of the target message within the `.proto` file. If someone reorders messages or adds one above your target, the index silently shifts and deserialization breaks.

This plugin solves that by generating a self-contained proto with the target message **always at index 0**. No manually maintained proto copies. No hardcoded indices. Schema and subject are derived from the source proto and stay in sync via code generation.

## Usage

### 1. Annotate messages

```protobuf
import "redpanda/api/common/v1/schema_registry.proto";

message LLMProvider {
  option (redpanda.api.common.v1.schema_registry) = {
    subject: "redpanda.aigw.llm-providers-value"
  };

  string name = 1;
  LLMProviderType type = 2;
  // ...
}
```

### 2. Add to buf.gen.yaml

```yaml
version: v2
plugins:
  # Standard Go protobuf generation
  - remote: buf.build/protocolbuffers/go
    out: ./gen/go
    opt: paths=source_relative

  # Schema Registry schema generation
  - local: protoc-gen-go-sr
    out: ./gen/go
    opt: paths=source_relative
```

### 3. Generate

```bash
buf generate
```

This produces `llm_provider_sr.go` alongside `llm_provider.pb.go`:

```go
// Code generated by protoc-gen-go-sr. DO NOT EDIT.
package aigwv1alpha1

const LLMProviderSRSchema = `// Generated by protoc-gen-go-sr. DO NOT EDIT.
syntax = "proto3";
package redpanda.api.aigw.v1alpha1;
import "google/protobuf/timestamp.proto";

message LLMProvider {
  string name = 2;
  string display_name = 3;
  LLMProviderType type = 4;
  ...
}

enum LLMProviderType {
  LLM_PROVIDER_TYPE_UNSPECIFIED = 0;
  LLM_PROVIDER_TYPE_OPENAI = 1;
  ...
}
`

const LLMProviderSRSubject = "redpanda.aigw.llm-providers-value"
```

### 4. Wire into kvstore

```go
import (
    commonkvstore "github.com/redpanda-data/common-go/kvstore"
    aigwv1alpha1 "your/proto/gen/package"
)

serde, err := commonkvstore.Proto(
    func() *aigwv1alpha1.LLMProvider { return &aigwv1alpha1.LLMProvider{} },
    commonkvstore.WithSchemaRegistry(
        srClient,
        aigwv1alpha1.LLMProviderSRSubject,
        aigwv1alpha1.LLMProviderSRSchema,
    ),
)
```

## What gets generated

The generated `.proto` string contains:

| Included | Excluded |
|----------|----------|
| Target message (index 0) | Services and RPCs |
| Referenced messages (transitively) | `buf.validate` annotations |
| Referenced enums | `google.api` annotations |
| Well-known type imports | Field behavior annotations |
| Reserved field declarations | Request/Response wrapper messages |
| Oneof groups | |
| Map fields | |

## Install

```bash
go install github.com/redpanda-data/common-go/protoc-gen-go-sr@latest
```

Or use as a local plugin in buf.gen.yaml:

```yaml
plugins:
  - local: ["go", "run", "github.com/redpanda-data/common-go/protoc-gen-go-sr"]
    out: ./gen/go
    opt: paths=source_relative
```
